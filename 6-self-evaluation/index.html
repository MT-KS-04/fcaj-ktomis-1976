Self-Assessment

Throughout my internship at the Workforce Bootcamp – First Cloud AI Journey program from May 5, 2026 to the present, I have had the opportunity to learn, practice, and apply knowledge acquired at university to a real-world technical environment on the AWS platform.

I completed 6 weeks of in-depth structured learning following the First Cloud Journey roadmap, covering the stages of Explore AWS Services, Migrate to AWS, and Optimize (Security & Operations). During this time, I independently studied, completed workshops, and wrote weekly worklogs on AWS services including IAM, VPC, EC2, S3, RDS, DynamoDB, CloudFront, CloudFormation, KMS, Secrets Manager, WAF, and more.

In terms of work ethic, I always strived to stay on schedule, proactively sought out resources when facing difficulties, and maintained a habit of writing detailed weekly worklogs to consolidate my learning.

Results Achieved

After 6 weeks of internship, I have achieved the following key results:

  • Understand foundational AWS Cloud concepts: account setup, basic security (MFA, IAM), and networking infrastructure (VPC, Subnet, Route Table, Security Group)
  • Proficient in using the AWS Management Console and AWS CLI to create and manage cloud resources
  • Understand the role of key services: S3, EC2, RDS, DynamoDB, CloudFront, Route 53, CloudWatch
  • Hands-on experience with Infrastructure as Code using CloudFormation and CDK for automated infrastructure deployment
  • Practiced Disaster Recovery strategies: Pilot Light and Warm Standby, understanding RTO/RPO and cost trade-offs
  • Applied advanced security practices: Permission Boundaries, IAM Conditions, KMS, Secrets Manager, WAF, Security Hub
  • Developed the ability to write structured technical worklogs consistently week over week
  • Gained practical insights into AI/Cloud trends through the FCAJ Community Day – May 23, 2026 event

Personal Strengths

Throughout the internship, I identified the following personal strengths:

  • Self-directed learning: Proactively planned and executed each week’s learning without waiting for explicit guidance
  • Patience with technical errors: When encountering errors (403 Forbidden S3, CLI credential issues, ResourceExistsException in Secrets Manager…), I investigated root causes and documented lessons to avoid repetition
  • Systematic thinking: Approached learning from foundational to advanced topics, focusing on 1–2 services per day to build solid understanding rather than rushing
  • Documentation skills: Maintained detailed worklogs with a consistent structure, including tasks performed, results, difficulties encountered, and lessons learned
  • Curiosity-driven mindset: Always asked “why” instead of just following instructions — for example, deliberately testing failure cases to understand underlying mechanics

Lessons Learned

Through the internship and event participation, I have drawn the following important lessons:

  • Theory must go hand-in-hand with practice: Reading documentation alone is not enough; only by working through workshops do you discover gaps in your understanding
  • Security must be a priority from the start: MFA, IAM least privilege, keeping credentials out of code — these must be set up immediately, not deferred
  • Cloud cost management is a required skill: Setting up AWS Budgets and cleaning up resources after each lab is essential to prevent unexpected charges
  • Security is multi-layered, not a single point: Access control, encryption, application protection, and monitoring must all be applied together
  • Context and product thinking matter more than pure technical skills: Learned from FCAJ Community Day — knowing how to prioritize correctly and focus on real value is what creates meaningful impact

Areas for Improvement

  • Time discipline: Learning pace was uneven across days; need to build a more consistent daily routine
  • Independent problem-solving: Still somewhat reliant on provided workshop guides; need to strengthen the ability to design solutions from scratch without scaffolding
  • Technical communication: Need to improve the ability to explain complex technical concepts in simpler, more accessible language for non-technical audiences
  • Real-world team collaboration: Most of the internship involved individual self-study; need to engage more in group projects to develop teamwork and coordination skills

Overall Assessment

Overall, I assess that I have successfully completed the first phase of my internship with good results. I have gained significant knowledge of AWS Cloud — from foundational services to more advanced topics such as security, Infrastructure as Code, and Disaster Recovery.

Although there are still areas to improve — including time discipline, independent problem-solving, and technical communication — I have shown clear week-over-week progress: from having no cloud background to being able to independently practice and explain foundational to intermediate AWS concepts. This forms a solid foundation for me to continue growing confidently in the cloud computing and AI field going forward.
