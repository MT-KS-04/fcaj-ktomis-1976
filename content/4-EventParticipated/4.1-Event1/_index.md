## 4. Main Content

### 4.1. AI & LLM Knowledge Sharing

- Updates on the latest trends in Large Language Models (LLMs) and enterprise deployment strategies
- Introduction to **Amazon Q** – an AI-powered tool supporting the full Software Development Lifecycle (SDLC)
- Clarification of **Context**, **AI Memory**, and **MCP (Model Context Protocol)** roles in enhancing AI system performance
- Understanding the reasoning mechanism of LLMs and the non-deterministic nature of output generation

### 4.2. Multi-Agent Systems Architecture

- How to design and build **Multi-Agent systems** for handling complex enterprise-level tasks
- Coordinating multiple AI agents to solve problems that a single model cannot handle alone
- **Guardrails and Permission Boundaries**: Ensuring safety and access control in enterprise AI systems

### 4.3. Hands-on Workshop Exercises

Participants practiced building enterprise-grade AI systems through 5 structured exercises:

1. **Add Authentication** – Implement Cognito/OAuth2 with JWT validation
2. **Implement Guardrails** – Configure Bedrock Guardrails for content filtering
3. **Permission Boundaries** – Set up IAM with least-privilege access
4. **MCP Integration** – Add Model Context Protocol for tool integration
5. **Terraform Deployment** – Create Infrastructure as Code for reproducible deployment

### 4.4. Hackathon Experience & Product Mindset

- Team UTMorpho built a fully functional product in approximately **18 hours**
- Key factors: **time management**, **clear task allocation**, and **leveraging AI tools** to accelerate development
- Demonstrated that AI should be treated as an **acceleration tool**, not a replacement, in the software process

### 4.5. Career Guidance & Industry Insights

- Hands-on experience from engineers working at leading Cloud/AI companies
- Essential skills for Cloud/AI engineers: product thinking, self-learning, and fast adaptability
- Practical perspectives on career paths in the information technology industry

---

## 5. Key Takeaways

| Topic                            | Insight                                                                               |
| -------------------------------- | ------------------------------------------------------------------------------------- |
| **Context is key**               | LLM output quality heavily depends on prompt design and context management            |
| **Security by design**           | Integrate Guardrails and Permission Boundaries from the start of AI system design     |
| **MCP unlocks extensibility**    | Model Context Protocol enables AI agents to interact with any external tool or system |
| **Product thinking > pure tech** | Prioritizing features around user value is what ultimately determines success         |
| **IaC is essential**             | Terraform enables reproducible, consistent AWS infrastructure deployment              |
