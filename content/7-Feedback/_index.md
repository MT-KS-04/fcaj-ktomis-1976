> Some thoughts and reflections after taking part in the First Cloud AI Journey program and building the final project with my team.

Throughout the program I went from the early weeks of learning AWS fundamentals to finally pulling everything together into a project that actually ran. It gave me a much clearer picture of cloud, serverless architecture, and how a real application is built and operated on AWS. Below are a few personal reflections.

### Overall Evaluation

**1. Learning and working environment**
The environment fits students who are new to cloud pretty well. The content goes from easy to hard - fundamentals first, then hands-on practice and the project - so it never felt overwhelming. I liked how self-directed it was: reading the docs myself, following the steps, and noting down the errors I hit, which gradually got me used to self-study and solving problems on my own. A few regular group check-ins where people share progress and ask each other questions would make it even better.

**2. Support from mentors / team admin**
The mentors and admins were helpful and their guidance was clear, so it was easy to follow. What I appreciated most was that when I got stuck on the Console or CLI, they'd point me toward the cause so I could find it myself rather than just handing over the answer. Harder in the moment, but it made me understand things more deeply and build the habit of debugging on my own instead of following steps mechanically.

**3. Relevance to my major**
The content lined up well with my IT major. Things like backend, APIs, databases, user authentication, file storage, and system deployment all tie directly into what I study at school. The final project in particular let me bring a lot of scattered knowledge into one place: building a backend API, working with a NoSQL database, connecting the frontend to the backend, handling file uploads, and wiring several cloud services together.

**4. Learning and skill development**
On the technical side, I got real hands-on practice with a range of services: S3 for file storage, Lambda for backend logic, API Gateway for the REST API, DynamoDB for the data, Cognito for sign-in, and Secrets Manager to hide sensitive info instead of hardcoding it. CloudWatch was open almost constantly for checking logs when something broke. On the soft-skill side, I improved at self-study, teamwork, report writing, and - most of all - calmly analyzing errors instead of panicking like I did at the start.

**5. Culture and team spirit**
Doing the project as a team showed me how much clear task-splitting matters. Each person owned a piece - frontend, backend, the AI part, deployment, testing - and it all had to line up. When something broke, the whole team dug in together to find the cause. That taught me a complete system isn't any one person's work, but many pieces that have to fit together.

---

### Additional Questions

**What satisfied me most**
The most satisfying part was building it myself and watching the system run smoothly from end to end. I used to know cloud only in theory; after finishing the project I finally understood how AWS services connect into a complete system. The moment everything clicked and actually worked felt great.

**What I think the program should improve**
I think the program could add a few things to make it easier for newcomers:

- An initial orientation on how to read AWS documentation so people don't get lost.
- Sample architecture diagrams for each type of project to picture things beforehand.
- A list of common errors like AccessDenied, CORS, and Lambda timeout, with fixes.
- A mid-term review to check on project progress.
- A resource-cleanup checklist for when you're done, to avoid forgetting and racking up costs.

**Would I recommend it to a friend?**
Yes. For anyone wanting to learn AWS, serverless, and how to build a real project, it's a solid stepping stone - you get both the fundamentals and a project for your portfolio. The one thing I'd warn them about is to be ready for a lot of self-study, since AWS is broad and the errors can be hard to make sense of when you're new.

---

### Suggestions and Expectations

A few small suggestions for a smoother experience: a clear week-by-week roadmap, more docs that explain architecture with diagrams, an environment-setup checklist and a security checklist before starting a project, plus some regular Q&A sessions with mentors.

As for me, after this project I want to dig deeper into serverless best practices, securing APIs with Cognito, Secrets Manager, event-driven design with SQS, CI/CD on AWS, and cost optimization when running a real system.

### Conclusion

Looking back, this was time well spent. From only knowing cloud on paper, I can now stand up a real system on AWS and understand what each piece does. There are still a few things that could be better, but the program gave me a realistic sense of how work gets done and how a tech project comes together. I want to thank the mentors, the team admin, and the program for the chance to learn, practice, and finish the project with my team. If the opportunity comes up, I'd like to stay with the community and help out the next batch however I can.
